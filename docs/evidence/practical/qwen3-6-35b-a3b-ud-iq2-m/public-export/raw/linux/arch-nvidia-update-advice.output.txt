### 2. Why Partial Upgrades Are Dangerous
*   **Never** update only specific packages (e.g., `sudo pacman -Syu nvidia`).
*   **Reason:** Arch Linux is a rolling release. Packages depend on each other. Updating only NVIDIA drivers while leaving the kernel, X11/Wayland libraries, or system utilities on older versions can cause dependency conflicts, broken libraries, or a non-booting system. Always update the entire system together.

### 3. NVIDIA & Wayland Specific Cautions
*   **Driver Consistency:** Ensure your `nvidia` driver package matches your running kernel version. If you update the kernel, you must also update the NVIDIA driver. Mismatched versions will cause the GPU to fail to initialize.
*   **Wayland Compatibility:** NVIDIA’s Wayland support has improved but can still be sensitive. After updating:
    *   Check if your compositor (KDE Plasma, GNOME, Sway) requires a restart or re-login.
    *   Be aware that some NVIDIA proprietary features may behave differently on Wayland vs. X11. If you encounter flickering or black screens, switching to X11 temporarily is a safe fallback.
*   **Secure Boot:** If Secure Boot is enabled, ensure your NVIDIA kernel modules are signed. After a kernel update, you may need to re-sign modules or reboot to allow the new kernel to load the driver.
